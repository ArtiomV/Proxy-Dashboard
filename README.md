# Proxy Dashboard

Мультитенантная платформа для управления мобильными прокси (4G/LTE) с полным биллингом, документооборотом и интеграцией с Точка Банком.

Система позволяет управлять парком USB-модемов, подключённых к нескольким серверам ProxySmart, предоставлять клиентам доступ через HTTP/SOCKS5, автоматически тарифицировать трафик и генерировать закрывающие документы.

## Ключевые возможности

- **Мульти-серверная архитектура** — управление неограниченным числом серверов ProxySmart (S1, S2, S3...) из единой панели
- **Автоматический биллинг** — ежедневные списания по трафику (руб/ГБ), за модем или фиксированная подписка
- **Банковская интеграция** — Webhook Точка Банка, автоматический зачёт платежей по ИНН
- **Документооборот** — генерация актов и счетов через Точка API, подписание, скачивание PDF
- **Клиентский портал** — личный кабинет с реквизитами прокси, аналитикой, биллингом
- **Мониторинг модемов** — статусы, uptime, speedtest, латентность, IP-история
- **Авто-восстановление** — автоматический USB-reset офлайн модемов (до 3 попыток)
- **Реферальная система** — клиенты получают 15% комиссии за приведённых

### Принцип работы

1. При старте загружает клиентов, сессии, настройки из SQLite в память
2. Мутации пишутся атомарно в SQLite (prepared statements + transactions) и обновляют in-memory cache
3. Серверы ProxySmart опрашиваются каждые 30 секунд (статусы) и 3 минуты (трафик/uptime)
4. Трафик агрегируется почасово и сохраняется в `traffic_hourly` с cross-validation двух счётчиков
5. Банковские платежи приходят через webhook, матчатся по ИНН и зачисляются автоматически

## Управление клиентами

### Профили
- Уникальный логин, хеширование пароля (bcrypt)
- Привязка к порту прокси-сервера (`portName`) — один клиент = один или несколько портов
- Поддержка привязки нескольких модемов к одному клиенту
- Контактная информация, заметки
- API-ключ для программного доступа
- Реферальная система (уникальный код, баланс комиссий 15%)
- Тип клиента: юр. лицо / физ. лицо
- Возможность приостановки биллинга

### Модели биллинга

| Тип | Описание |
|-----|----------|
| `per_gb` | Оплата за потреблённый трафик (руб/ГБ). Цена зависит от количества модемов (тарифная сетка) |
| `per_modem` | Фиксированная ежемесячная плата за каждый привязанный модем |
| `flat` | Фиксированная ежемесячная подписка независимо от потребления |

### Реквизиты для документов
- ИНН, КПП, юридическое название, адрес
- Информация о договоре
- Авто-генерация актов и счетов (переключатели per client)

### Баланс
- Атомарные операции с балансом (`atomicCredit` / `atomicDebit`) через SQLite transactions
- Полный журнал (ledger) с balance_before/balance_after для каждой операции
- Поддержка нескольких валют (по умолчанию RUB)
- Ежедневные автоматические списания по тарифу

## Управление прокси и модемами

### Прокси-серверы
- Мульти-серверная архитектура — каждый сервер (S1, S2...) имеет URL, логин, пароль, публичный IP, страну, таймзону
- Динамическое обнаружение серверов из переменных окружения `API_S*_*`
- Кеширование данных серверов (TTL 30 сек) с deduplication параллельных запросов
- Поддержка HTTP и HTTPS подключений к ProxySmart API

### Модемы
- **Статусы**: online / rotating / rebooting / offline
- **Информация**: IMEI, модель, оператор (автонормализация), телефонный номер
- **Сеть**: тип подключения (LTE/3G/2G), уровень сигнала, внешний IP
- **Управление**: сброс IP, перезагрузка, USB-reset, отправка SMS, USSD-запросы
- **OS Spoofing**: массовая смена OS fingerprint (Windows/Mac/Linux/Android/iOS)
- **Speedtest**: тестирование скорости через cURL с историей результатов и повторными тестами

### Авто-восстановление (Auto-Recovery)
Когда модем уходит в офлайн более чем на 60 секунд, система автоматически пытается его восстановить:
- Отправляет USB-reset через ProxySmart API
- До 3 попыток с интервалом 3 минуты (совпадает с циклом опроса `trackModems`)
- Логирует каждую попытку: `[AutoRecovery] USB reset #1/3 for NICK (S1), offline 65s`
- При успешном восстановлении: `[AutoRecovery] NICK back online after 2 reset(s)`
- После 3 неудачных попыток прекращает (требуется ручное вмешательство)

### Мониторинг латентности
- Периодическая проверка доступности прокси через cURL (connect + total time)
- Хранение результатов в `proxy_checks` с агрегацией по никам
- Настраиваемый интервал проверки (по умолчанию 60 минут)

## Аналитика трафика

### Сбор данных
- Два источника: дневной счётчик (`BW_DAY`) и месячный (`BW_MONTH`) из ProxySmart
- Cross-validation: при расхождении >5% данные помечаются как `uncertain`
- Почасовая агрегация с инкрементальными обновлениями (не перезаписывает)
- Хранение в `traffic_hourly` с полями: server, port, nick, operator, client, hour, bytes_in, bytes_out

### Визуализация
- **Тепловая карта** — трафик по часам/дням (фильтр по клиенту/серверу/модему)
- **Дневные графики** — потребление по дням с навигацией по месяцам
- **Кастомный диапазон дат** — произвольный период для анализа
- **Top hosts** — самые посещаемые домены (глобально и per client)
- **Bandwidth reports** — отчёты по серверам за периоды

### Периоды аналитики (админ)
- Сегодня, Вчера, Месяц, Предыдущий месяц, Всё время
- Фильтрация по стране, клиенту, оператору

## Биллинг и платежи

### Ledger (журнал операций)
- Полная история всех финансовых операций клиента
- Типы: `charge` (списание), `payment` (оплата), `correction` (корректировка), `bank_payment`, `payment_reversal`
- Каждая запись: balance_before, balance_after, timestamp, amount, currency, GB трафика, кол-во модемов
- Пагинация и поиск по журналу

### Ежедневные списания
- Автоматический расчёт стоимости по тарифной модели клиента
- Учёт тарифной сетки (скидки за объём модемов)
- Возможность приостановки биллинга per client
- Логирование каждого списания с детализацией

### Платежи
- Ручное добавление через админку
- Автоматический зачёт из банковских платежей (по ИНН)
- Источник: manual, tochka_webhook, tochka_sync
- Реверсирование платежей с атомарным обновлением баланса

### Реферальная система
- Каждый клиент получает уникальный реферальный код
- При оплате привлечённым клиентом, реферер получает 15% комиссии
- Комиссии начисляются автоматически при банковских платежах
- Отдельный баланс `referral_balance` для вывода комиссий

## Документооборот

### Акты (Closing Documents)
- Автоматическая генерация в конце месяца для всех клиентов с `autoActs = true`
- Детализация услуг: трафик (ГБ), стоимость за ГБ, период
- Отправка в Точка Банк для подписания
- Скачивание подписанного PDF
- Нумерация актов с инкрементом

### Счета (Bills)
- Автоматическая генерация для выставления клиентам
- Интеграция с Точка API для отправки
- Отслеживание статуса: unpaid / paid / sent

### Загружаемые документы
- Загрузка произвольных файлов для клиента (договоры, сканы и т.д.)
- Скачивание клиентом из ЛК

## Интеграция Точка Банк

### Webhook входящих платежей
1. Точка отправляет POST на `/api/tochka/webhook`
2. Верификация подписи RSA-SHA256 через JWKS
3. Парсинг: ИНН плательщика, сумма, назначение, paymentId
4. Дедупликация по paymentId (защита от повторной обработки)
5. Матчинг по ИНН -> клиент
6. Автоматическое зачисление на баланс + запись в ledger
7. Начисление реферальной комиссии при наличии реферера

### Синхронизация
- Ручная синхронизация исторических платежей из Точка API
- Регистрация webhook через API

### Генерация документов
- Формирование актов с детализацией услуг
- Формирование счетов
- Русские названия месяцев
- Поддержка ИП и юр. лиц
- Скачивание PDF через Точка API

## Безопасность

- **Аутентификация**: сессионные токены с TTL (24 часа), заголовок `X-Auth-Token`
- **Хеширование**: bcrypt для паролей клиентов
- **Rate limiting**: настраиваемый лимит попыток входа, ограничение API-запросов
- **Helmet**: HTTP security headers (CSP, HSTS, X-Frame-Options)
- **Мультитенантность**: строгая фильтрация по `portName` — клиент видит только свои данные
- **Аудит**: полный лог действий администратора с IP-адресом (хранение 90 дней)
- **Импersonation**: администратор может войти от лица клиента (логируется в аудит)
- **Webhook верификация**: RSA-SHA256 проверка подписи Точка Банк
- **SQL Injection**: все запросы через prepared statements
- **XSS**: escapeHtml для всего пользовательского контента
- **CSRF**: проверка Referer/Origin для мутирующих запросов

## Админ-панель (admin.html)

Одностраничное приложение (SPA) с вкладками:

### 1. Модемы
- Таблица всех модемов со статусами (online/offline/rotating/rebooting)
- Группировка по странам/серверам
- Управление: сброс IP, перезагрузка, USB-reset
- SMS и USSD через модем
- Speedtest с историей
- Детальная карточка модема (IP, оператор, модель, сигнал, uptime)

### 2. Аналитика
- Тепловая карта трафика (часы x дни)
- Дневной трафик по клиентам/модемам

## Конфигурация

### Тарифная сетка (settings)

```json
{
  "priceTiers": [
    { "minModems": 1, "maxModems": 29, "price": 4500 },
    { "minModems": 30, "maxModems": 49, "price": 4275 },
    { "minModems": 50, "maxModems": 99, "price": 4050 },
    { "minModems": 100, "maxModems": 999, "price": 3825 }
  ],
  "speedTestTimes": ["02:00", "14:00"],
  "proxy_check_interval_min": 60
}
```

### Автоматизация (фоновые процессы)
- **Опрос модемов**: статусы каждые ~30 сек (кеш ProxySmart)
- **Трафик и uptime**: каждые 3 минуты (`trackModems`)
- **Почасовая агрегация**: каждые 10 минут (`aggregateHourlyTraffic`)
- **Speedtest**: по расписанию (например, 02:00 и 14:00)
- **Проверка латентности**: каждые 60 минут
- **Синхронизация ротации**: каждые 30 минут
- **Очистка аудита**: записи старше 90 дней
- **Авто-восстановление**: USB-reset офлайн модемов (в цикле trackModems)

## Деплой (продакшн)

```bash
# На сервере
scp -r project/* user@server:/path/to/app/
ssh user@server "cd /path/to/app && npm install && pm2 restart proxy-dashboard"
```

### Требования
- Node.js >= 18
- PM2 (рекомендуется для продакшн)
- cURL (для speedtest и проверки латентности)

### БД
- Автоинициализация при первом запуске через `schema.sql`
- WAL mode для конкурентных записей
- Foreign keys включены
- Safe ALTER TABLE для эволюции схемы
- Backup: `GET /api/admin/backup` скачивает SQLite файл
