# Proxies.Rent — Платформа управления мобильными прокси

Полнофункциональная платформа для управления мобильными прокси на базе Node.js. Включает личный кабинет клиента, административную панель, автоматический биллинг, публичный API, мониторинг модемов и систему аналитики.

---

## Возможности

### Личный кабинет клиента

**Панель управления**
- Таблица модемов в реальном времени: статус (онлайн/офлайн/ротация), IP-адрес, оператор, страна
- HTTP и SOCKS5 реквизиты с кнопками копирования
- Сброс IP одним кликом + прямые ссылки для API-ротации
- Статистика трафика: сегодня / вчера / за месяц (in/out)
- Сводные карточки: количество портов, трафик, баланс, аптайм
- История IP-адресов по каждому модему
- Экспорт реквизитов в TXT и CSV
- Детекция офлайн-модемов — отключённые модемы продолжают отображаться с меткой OFFLINE

**История баланса**
- Карточки: текущий баланс, расход за месяц, всего оплачено, всего списано
- Таблица транзакций: дата, тип, сумма, трафик, баланс после, примечание
- Фильтр по месяцам
- Автоматическое добавление текущего месяца

**Документы**
- Список документов клиента (договоры, акты)
- Скачивание документов

**API-документация**
- Персональный API-ключ с кнопкой копирования
- Полная документация эндпоинтов с примерами
- Примеры кода на cURL, Python, JavaScript
- Справочник ошибок

**Партнёрская программа**
- Уникальный реферальный код
- Отслеживание привлечённых клиентов
- Статистика заработка (15% от платежей рефералов)

**Инструменты**
- Массовая проверка прокси (до 50 за раз) — определение рабочего IP, время отклика, статус
- Конвертер форматов прокси (`ip:port:login:pass` ↔ `login:pass@ip:port` и др.)

### Административная панель

**Управление клиентами**
- CRUD-операции над клиентами
- Настройка тарифа: per_gb (за гигабайт) или per_modem (за модем/месяц)
- Управление платежами: добавление, удаление, ручная корректировка баланса
- Просмотр детального ledger-лога клиента
- Перегенерация API-ключей
- Загрузка и удаление документов

**Управление модемами**
- Просмотр статуса всех модемов
- Сброс IP / перезагрузка / USB-reset
- Хранение и применение конфигураций модемов
- SMS/USSD-команды

**Управление портами**
- Создание, обновление, привязка к клиенту, удаление портов
- Обновление логин/пароля
- Просмотр свободных портов

**Мониторинг скорости**
- Автоматические спидтесты (настраиваемое расписание)
- Ручной запуск тестов
- История замеров: download/upload (Mbps), ping (ms)
- Автоматический ретест при аномально низких скоростях

**Аналитика трафика**
- Топ посещаемых хостов (агрегация по портам)
- Отчёты по трафику за период
- Подсчёт уникальных IP
- Сброс счётчиков

---

### Система биллинга

- **Два типа тарификации:**
  - `per_gb` — цена × потреблённые гигабайты
  - `per_modem` — фиксированная цена за модем в месяц (пропорционально дням)
- **Ежедневное автоматическое списание** (настраиваемое время)
- **Дельта-подход** — учитывает только новый трафик с последнего снапшота
- **Устойчивость к сбросу месяца** — корректно обрабатывает обнуление счётчиков ProxySmart
- **Персистентный баланс** — баланс никогда не пересчитывается, только модифицируется
- **Append-only ledger** — полный аудит-трейл всех операций (списания, оплаты, корректировки)
- **Catch-up при рестарте** — если сервер пропустил списание, оно выполняется при запуске

### Публичный API

**Аутентификация:** заголовок `X-API-Key` или параметр `?apikey=`

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| `GET` | `/api/v1/proxy` | Все прокси клиента (JSON) |
| `GET` | `/api/v1/proxies` | Прокси в разных форматах (JSON, TXT, CSV) |
| `GET` | `/api/client/reset_ip_by_token` | Сброс IP по токену (без авторизации) |
| `GET` | `/api/docs` | Документация API |

**Пример ответа `/api/v1/proxy`:**
```json
{
  "success": true,
  "proxies": [
    {
      "name": "MD2_43",
      "operator": "Moldtelecom",
      "country": "Moldova",
      "http": "89.149.100.92:8029",
      "socks5": "89.149.100.92:5029",
      "login": "user123",
      "password": "pass456",
      "external_ip": "178.17.42.105",
      "is_online": true,
      "reset_ip_url": "https://dashboard.proxies.rent/api/client/reset_ip_by_token?token=..."
    }
  ],
  "billing": {
    "balance": 5000,
    "currency": "RUB"
  }
}
```

### Мониторинг и трекинг

- **IP-трекинг** — запись каждой смены IP с таймстемпом (каждые 10 мин)
- **Uptime-трекинг** — процент аптайма по каждому модему
- **Персистентность офлайн-модемов** — `known_modems.json` сохраняет информацию об отключённых модемах
- **Кеширование данных** — при недоступности API-сервера используется кеш с пометкой

## Структура данных

| Файл | Назначение |
|------|------------|
| `clients.json` | Аккаунты клиентов, API-ключи, балансы, тарифы |
| `billing_ledger.json` | Журнал списаний и пополнений |

| `server_cache.json` | Кеш последних данных серверов |
| `ip_tracking.json` | Текущий IP каждого модема |
| `uptime_tracking.json` | Статистика аптайма |
| `ip_history.json` | Исторические IP-адреса с таймстемпами |
| `speedtest_history.json` | Результаты спидтестов |
| `top_hosts_cache.json` | Агрегированные топ-хосты |
| `documents/` | Документы клиентов |

## Конфигурация

Файл `.env` в корне проекта:

```env
# Порт сервера
PORT=3000

# API-серверы ProxySmart (поддержка нескольких)
API_S1_URL=http://server1-ip:port
API_S1_USER=proxy
API_S1_PASS=proxy
API_S2_URL=http://server2-ip:port
API_S2_USER=proxy
API_S2_PASS=proxy

# Логины модемов
MODEM_LOGIN_MD2_64=login123

# Аккаунты пользователей
# Формат: USER_<login>=<password>|<portNameFilter>
# * = администратор (видит всё)
USER_admin=secretpass|*
USER_client1=clientpass|MD2
```

## Автоматические задачи

| Задача | Расписание | Описание |
|--------|------------|----------|
| IP-трекинг | Каждые 10 мин | Проверка смены IP, обновление аптайма |
| Ежедневный биллинг | Настраиваемое (по умолчанию 00:00 UTC) | Списание за трафик/модемы |
| Спидтесты | Настраиваемое (по умолчанию 02:00, 14:00 UTC) | Замер скорости всех модемов |
| Агрегация хостов | 03:00 UTC | Сбор статистики посещаемых доменов |
| Авто-создание клиентов | При обнаружении новых портов | Автоматическая регистрация |

## Установка и запуск

```bash
# Клонировать репозиторий
git clone https://github.com/ArtiomV/Bandwidth.git
cd Bandwidth

# Установить зависимости
npm install

# Настроить окружение
cp .env.example .env
# Отредактировать .env — указать серверы ProxySmart и аккаунты

# Запустить
node server.js

# Для production
nohup node server.js > server.log 2>&1 &
```

Дашборд будет доступен по адресу `http://localhost:3000`
