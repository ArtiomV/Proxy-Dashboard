# ADR: иерархия источников данных по трафику

**Статус:** принято (2026-07, WP3.1)
**Контекст:** пять хранилищ трафика исторически смешивались: `daily_traffic` (SQL),
`dailyTraffic` (память), `traffic_hourly`, `hourly_snapshots`, live-счётчики
ProxySmart. Чтения шли из разных мест с расхождениями.

## Решение (канон)

| Источник | Роль | Читатели |
|---|---|---|
| `traffic_hourly` | Детальный почасовой ряд. Heatmaps, почасовая аналитика, ACL-история атрибуции (`client_name`) | `src/routes/analytics.js`, `src/modems/ownership.js`, портал (почасовые графики) |
| `daily_traffic` | **Канон дневных сумм и биллинга.** MAX-семантика при записи (значение не может уменьшиться) | `runDailyBilling` (durable-путь), `syncYesterdayTraffic`, портал (дни) |
| Live-счётчики ProxySmart | Только «сегодня, прямо сейчас». Биллинг берёт MAX(durable, live) только в день начисления — на случай рестарта ProxySmart, обнулившего вчерашние счётчики | `computeClientYesterdayBytes`, виджеты |
| `dailyTraffic` (память) | **Read-through кэш** таблицы `daily_traffic`, а не независимый источник. Любая запись идёт через `_dtUpsert` + обновление кэша в той же точке | быстрые читатели без SQL |
| `hourly_snapshots` | Технические снапшоты (trend/агрегация), не биллинг | фоновые джобы |

## Правила записи

1. Все писатели в `daily_traffic` используют MAX-семантику: `max(existing, new)`
   (ProxySmart-рестарт не должен обнулить уже записанное).
2. Дата ключа — **MSK-день** для всех серверов (один канонический день; см.
   комментарий в `syncYesterdayTraffic` — пер-серверная полночь ломала читателей).
3. Атрибуция клиента (`portName`) пишется вместе с байтами там, где схема
   позволяет; историческая атрибуция — через `traffic_hourly.client_name`.

## Что НЕ сделано (и почему)

- Перевод биллинга целиком на `daily_traffic` без MAX с live: отложено до
  недели наблюдений за `balance-reconcile` (WP5) на проде — дрейф до/после
  должен быть измерен, а не предположен. Это прямое требование ТЗ (WP3,
  высокий риск — деньги).
- `client_name` в `daily_traffic` (миграция схемы): приедет вместе с
  переводом писателей на единую `recordDailyTraffic()`.
