10.Поведение и обновление данных

  • Оркестратор — renderAccNew(): вызывается при входе на вкладку и каждые 10 с из общего цикла loadData().
  • Троттлинг финансов: /api/admin/finance_dashboard и график MRR обновляются не чаще раза в ~55 с; между тиками пересобирается только дешёвый DOM. Chart.js-инстансы не destroy/create на каждый тик — иначе видимое мигание.
  • Раскрывашки: открытые по умолчанию секции наполняет сам renderAccNew (guard dataset.loaded); закрытые грузятся при первом открытии (toggle-обработчик), «Ротации·IP·ёмкость» и «Обращения к API» перезагружаются при каждом открытии.
  • Состояние переживает автообновление: открытость details, выбранный период инфраструктуры (_NEW_INFRA_DAYS), фильтры API (_apiAccessState), вид/чип хитмапа — всё в JS-глобалях; рендер не должен их сбрасывать.
  • Клики: плитки внимания → попап списка модемов; ⏻/⟳ в «Топ проблемных» → экшены с тостами; строки клиентов некликабельны (просмотр).
  • Пустые состояния: каждая карточка обязана иметь текст-заглушку («Нет данных», «Загрузка…») 11–12px text-3 — контейнеры не схлопывать.
