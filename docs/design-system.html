Дизайн-система Arendaproxy v2

Переносимая спецификация: токены, типографика, компоненты и правила вёрстки, достаточные для воспроизведения дизайна в другом проекте с нуля. Эта страница сама свёрстана по описываемой системе — каждый образец «живой», рядом — CSS для копирования.

версия 2.0 · 3 июля 2026
0 · Философия

Это плотный операционный UI, который сканируют, а не читают. Пять принципов, из которых выводится всё остальное:

1 · Две темы по назначению. Операционные страницы (парк устройств, настройки) — тёмные; денежные и обзорные (дашборд, финансы) — светлые. Метрики (радиусы, отступы, шрифты) общие, различаются только цвета.

2 · Белые карточки на сером поле. Вся информация живёт в карточках r12 с волосяной границей и еле заметной тенью. Фон страницы никогда не белый.

3 · Таблицы без линий. Ни рамок, ни разделителей строк — только «зебра» чётных строк и шапка-полоса со скруглёнными краями.

4 · Семантический цвет ≠ акцент. Синий акцент — для навигации и действий. Состояния — всегда зелёный/янтарный/красный, и почти всегда на тонированной подложке (10–12% прозрачности), а не сплошной заливкой.

5 · Цифры — моноширинные. Всё числовое — JetBrains Mono или font-feature-settings:"tnum", чтобы колонки цифр стояли ровно.

Топ проблемных модемов

Контент секции — таблица или сетка. Открытые по умолчанию получают атрибут open, содержимое ленивых грузится по первому раскрытию.

Секционные лейблы и иконки

Пульс бизнеса

Страница дашборда делится не заголовками, а лейблами-линейками: серый uppercase-текст 11px/700/.7px + серая SVG-иконка 14px, справа линия 1px до края. Иконки — только инлайн-SVG (никаких эмодзи и icon-фонтов): viewBox 24×24, fill:none; stroke:currentColor; stroke-width:1.8; stroke-linecap/join:round; в навигации 18px, в заголовках 14–16px. Цвет наследуется от текста; акцентные — color:var(--accent).

Графики (Chart.js)

Единый вид: сетка только горизонтальная rgba(0,0,0,.07) (тёмная тема: rgba(255,255,255,.06)), без осевых линий (border.display:false) и без тиков; подписи осей #6b7280, 10px. Бары: borderRadius:5, borderSkipped:false, maxBarThickness:42, barPercentage:.6 (у плотных 60-дневных — borderRadius:2). Легенда: либо HTML-точки 8px в шапке карточки, либо Chart.js снизу c usePointStyle:'circle', boxWidth:6. Тултип: белый, r6, с итоговой строкой «Итого». Высоты: главный график 230px, дневной 300px (мобайл 220px).

5 · Каркас приложения

Левый сайдбар (десктоп ≥769px)

Фикс-панель 208px, фон bg-1, правая граница 1px. Три яруса, разделённые линиями:

1 · Бренд: лого 24px + название 14px/700/−.2px.
2 · Навигация (flex:1): пункты — строка «иконка 18px + текст 13px», r8, паддинг 9px 12px, gap 2px между пунктами; hover bg-2; активный — accent-dim фон, accent текст/иконка, вес 600 и полоска 2.5px слева.
3 · Футер: версия (10px mono, text-3) слева · тема · уведомления · выход. Без часов и статистики.

Страницы и шапки

Контент: паддинг 18px 26px, одна колонка карточек. Двухъярусная шапка страницы: ряд 1 — заголовок 16px/600 слева, поиск-пилюля + первичная кнопка справа, разделитель снизу; ряд 2 — чипы-фильтры со счётчиками слева, селекты/тогл вида справа. Суб-навигация внутри страницы (Финансы, Настройки) — свой мини-сайдбар ~200px с группами (лейбл 11px caps) и пунктами как в главном сайдбаре.

Модалки: r12, тень 0 24px 64px rgba(0,0,0,.5), шапка 14px 24px с заголовком 14px/600 и ×, вкладки — underline-табы; ширина 660–740px, max-width 95vw.

6 · Мобильная адаптация

Правило №1: контент идентичен десктопу — ничего не скрывается, всё адаптируется. Два брейкпоинта: ≤768px (уходит сайдбар) и ≤480px (телефон).

ПаттернКак
Навигациясайдбар → верхний бар; на ≤480 — нижний таб-бар (fixed, 6 пунктов «иконка над подписью», body padding-bottom 58px)
Широкие таблицыникогда не резать: обёртка overflow-x:auto; -webkit-overflow-scrolling:touch у карточки/тела секции
Реестры-таблицы (парк устройств)на ≤480 CSS-карточкизация: thead скрыт, tr → карточка flex-column, каждый td → строка «лейбл: значение» через td::before{content:attr(data-label)}; статус-пилюля и заголовок наверх через order; индикатор проблемы — цветная левая кромка 4px
Сеткиminmax(min(Npx,100%),1fr) — схлопываются сами; KPI-ряды 4→2 колонки; hero 4→2×2
Шапки страницflex-wrap: поиск на всю ширину второй строкой, инпутам width:100%!important
Модалкиwidth:95vw!important; max-height:90vh; полоса вкладок — в overflow-x:auto
Плотностьпаддинги страниц 24px→12px; кегль таблиц −1px; графики −30% высоты
